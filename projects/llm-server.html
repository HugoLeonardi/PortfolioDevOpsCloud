📌 Présentation du projet

Ce projet déploie un serveur IA self-hosted sur une machine physique sous Ubuntu 24.04, pensé pour l'usage d'un proche, professionnel en ingénierie CVC (chauffage, ventilation, climatisation). L'objectif est de disposer d'un assistant LLM local, souverain, sans dépendance à un service cloud tiers.

La stack est orchestrée avec Docker Compose et expose chaque service derrière un reverse proxy Caddy (HTTPS automatique) protégé par Authelia (SSO + 2FA). Le modèle LLM tourne via Ollama directement sur le GPU AMD (bare metal, ROCm).

  • Architecture modulaire : chaque service est isolé dans son propre conteneur
  • HTTPS en local avec certificat CA interne Caddy (tls internal)
  • Authentification centralisée via Authelia (cookie de session SSO)
  • Stockage cloud privé avec Nextcloud (MariaDB + Redis)
  • Ollama bare metal pour accès direct GPU sans overhead de virtualisation
ℹ️

Contexte : usage professionnel personnel. Domaine local llm.test (phase test) → vrai domaine + Let's Encrypt automatique en production. GPU : AMD RX 6950 XT 16 Go VRAM via ROCm 7.x.

🏗️ Architecture des services

La stack repose sur Docker Compose. Caddy est le seul point d'entrée exposé : il dispatch le trafic vers les services selon le sous-domaine, après vérification par Authelia. Ollama tourne en bare metal pour exploiter directement le GPU.

🔒 Sécurité

La stack est conçue pour être déployée en production avec un minimum de modifications. Les choix de sécurité sont documentés et distinguent clairement la phase de test de la configuration cible.

🔒
HTTPS partout (Caddy TLS)

En local : tls internal génère un CA local Caddy et des certificats signés automatiquement. En production, un simple changement de domaine active Let's Encrypt sans modifier la configuration applicative.

🔐
SSO centralisé — Authelia

Toutes les applications passent par Authelia. Un seul point d'authentification, une seule session. Configuration one_factor en test, TOTP (two_factor) prévu pour la production.

🌐
Domaine cookie-safe

Le domaine llm.test est utilisé à la place de localhost : les cookies de session Authelia nécessitent un domaine avec point. Configuré via /etc/hosts en local.

🗝️
Secrets dans .env (hors Git)

Tous les mots de passe et tokens (MariaDB, Nextcloud, Authelia JWT…) sont stockés dans un fichier .env non commité. Les hashs bcrypt sont générés via l'image officielle Authelia.

🔑
Mots de passe — Bcrypt

Les mots de passe Authelia sont stockés en hash bcrypt dans users_database.yml. Génération via l'image officielle : aucun mot de passe en clair dans les fichiers de configuration.

⚠️
2FA — À activer en production

La politique one_factor est utilisée pour simplifier les tests. En production : passer à two_factor (TOTP), activer ufw et restreindre les accès SSH.

Extraits de configuration clés

① Caddyfile — TLS interne + forward auth Authelia

caddyfile 
chat.llm.test {
  tls internal
  forward_auth authelia:9091 {
    uri /api/authz/forward-auth
    copy_headers Remote-User Remote-Groups Remote-Name Remote-Email
  }
  reverse_proxy open-webui:8080
}

② Génération d'un hash bcrypt pour Authelia

bash 
# Aucun mot de passe en clair dans les fichiers versionnés
docker run --rm authelia/authelia:latest \
  authelia crypto hash generate bcrypt \
  --password 'MonMotDePasse'

③ Authelia — politique d'accès par domaine

yaml 
access_control:
  default_policy: deny
  rules:
    - domain: 'auth.llm.test'
      policy: bypass
    - domain: '*.llm.test'
      policy: one_factor  # → two_factor en prod

⚙️ Services déployés

Open WebUI + Ollama

Open WebUI fournit une interface chat compatible avec l'API Ollama. Ollama tourne en bare metal (hors Docker) pour accéder directement au GPU AMD RX 6950 XT via ROCm — sans overhead de virtualisation, avec accès direct aux 16 Go de VRAM. Open WebUI se connecte à Ollama via localhost:11434 avec le mode réseau host.

Nextcloud + MariaDB + Redis

Cloud privé pour le stockage et partage de fichiers. MariaDB assure la persistance des données, Redis gère le cache et les verrous de fichiers pour les performances. Les trois conteneurs communiquent sur un réseau Docker interne, sans exposition directe.

Portainer

Interface de gestion des conteneurs Docker. Accessible uniquement via Authelia (sous-domaine portainer.llm.test), il permet de superviser l'état de la stack, consulter les logs et redémarrer des services sans passer par la CLI.

ℹ️

Modèles LLM recommandés avec le RX 6950 XT (16 Go VRAM) : Qwen 2.5 14B Q4_K_M (~9 Go, GPU-only), Mistral Nemo 12B (~7.5 Go, GPU-only), Qwen 2.5 32B Q4_K_M (~20 Go, GPU+RAM offload).

🔭 Évolutions prévues

  • Ollama + ROCm — Installer et configurer l'accès GPU bare metal (AMD ROCm 7.x) pour faire tourner les modèles LLM avec accélération matérielle.
  • WireGuard VPN — Accès distant sécurisé à la stack depuis l'extérieur, sans exposer de port sur Internet.
  • Production — Vrai domaine + Let's Encrypt automatique via Caddy, activation du 2FA TOTP Authelia, durcissement avec ufw.
  • Hermes Agent — Déploiement d'un agent IA autonome (NousResearch) connecté à Ollama pour des tâches automatisées.
⚠️

Passage en production : changer tls internal par le vrai domaine dans le Caddyfile (Let's Encrypt s'active automatiquement), passer Authelia en two_factor et migrer SQLite vers PostgreSQL.
