📌 Présentation du projet

Ce projet déploie un serveur IA self-hosted sur une machine physique sous Ubuntu 24.04, pensé pour l'usage d'un proche, professionnel en ingénierie CVC (chauffage, ventilation, climatisation). L'objectif est de disposer d'un assistant LLM local, souverain, sans dépendance à un service cloud tiers.

La stack est orchestrée avec Docker Compose et expose chaque service derrière un reverse proxy Caddy (HTTPS automatique Let's Encrypt) protégé par Authelia (SSO). Le modèle LLM tourne via Ollama directement sur le GPU AMD (bare metal, ROCm).

  • Architecture modulaire : chaque service est isolé dans son propre conteneur
  • HTTPS automatique via Caddy + Let's Encrypt sur domaine personnalisé
  • Authentification centralisée via Authelia (cookie de session SSO)
  • Stockage cloud privé avec Nextcloud (MariaDB + Redis)
  • Ollama bare metal pour accès direct GPU sans overhead de virtualisation
  • Installation entièrement automatisée via setup.sh
ℹ️

Contexte : usage professionnel personnel. Domaine alba-arietis.com avec certificats Let's Encrypt automatiques. GPU : AMD RX 6950 XT 16 Go VRAM via ROCm 6.4.

🏗️ Architecture des services

La stack repose sur Docker Compose. Caddy est le seul point d'entrée exposé : il dispatche le trafic vers les services selon le sous-domaine, après vérification par Authelia. Ollama tourne en bare metal pour exploiter directement le GPU.

🔒 Sécurité

🔀
HTTPS automatique — Let's Encrypt

Caddy gère automatiquement l'émission et le renouvellement des certificats TLS. Le port 80 reste ouvert uniquement pour le challenge ACME HTTP-01 — tout le trafic est redirigé vers HTTPS.

🔐
Authelia SSO — forward_auth

Chaque sous-domaine est protégé via le mécanisme forward_auth de Caddy. Aucun service n'est accessible sans session Authelia valide. Reset de mot de passe désactivé — gestion manuelle via users_database.yml.

🛡️
Pare-feu UFW

Seuls les ports nécessaires sont ouverts : 22 (SSH), 80 (ACME), 443 (HTTPS), 51820 (WireGuard). Ollama écoute sur toutes les interfaces en interne mais n'est pas exposé publiquement.

🗝️
Secrets dans .env (hors Git)

Tous les secrets (MariaDB, Nextcloud, JWT Authelia, clés de session) sont générés automatiquement avec openssl rand et stockés dans un .env exclu de Git. Les hashs bcrypt sont générés via l'image officielle Authelia.

📡
VPN WireGuard — Freebox

Accès distant au réseau local via WireGuard hébergé sur la Freebox Pop. Permet d'accéder au serveur sur son IP privée (192.168.2.56) sans exposer de ports supplémentaires.

🔑
Mots de passe — Bcrypt cost 12

Les mots de passe Authelia sont stockés en hash bcrypt (coût 12) dans users_database.yml. Génération via l'image officielle : aucun mot de passe en clair dans les fichiers versionnés.

Extraits de configuration clés

① Caddyfile — Let's Encrypt + forward auth Authelia

caddyfile 
# Caddy émet le certificat Let's Encrypt automatiquement
chat.alba-arietis.com {
  forward_auth authelia:9091 {
    uri /api/authz/forward-auth
    copy_headers Remote-User Remote-Groups Remote-Name Remote-Email
  }
  reverse_proxy open-webui:8080
}

② Génération d'un hash bcrypt pour Authelia

bash 
# Aucun mot de passe en clair dans les fichiers versionnés
docker run --rm authelia/authelia:latest \
  authelia crypto hash generate bcrypt \
  --password 'MonMotDePasse'

③ Authelia — politique d'accès par domaine

yaml 
access_control:
  default_policy: deny
  rules:
    - domain: 'auth.alba-arietis.com'
      policy: bypass
    - domain: '*.alba-arietis.com'
      policy: one_factor

⚙️ Services déployés

Open WebUI + Ollama

Open WebUI fournit une interface chat compatible avec l'API Ollama. Ollama tourne en bare metal (hors Docker) pour accéder directement au GPU AMD RX 6950 XT via ROCm 6.4 — sans overhead de virtualisation, avec accès direct aux 16 Go de VRAM. Open WebUI se connecte à Ollama via host.docker.internal:11434. Les modèles sont stockés sur un second disque NVMe dédié monté sur /data.

Nextcloud + MariaDB + Redis

Cloud privé pour le stockage et partage de fichiers. MariaDB assure la persistance des données, Redis gère le cache et les verrous de fichiers pour les performances. Les trois conteneurs communiquent sur un réseau Docker interne, sans exposition directe.

Portainer

Interface de gestion des conteneurs Docker. Accessible uniquement via Authelia sur portainer.alba-arietis.com, il permet de superviser l'état de la stack, consulter les logs et redémarrer des services sans passer par la CLI.

ℹ️

Modèles LLM utilisés avec le RX 6950 XT (16 Go VRAM) : Qwen 2.5 27B Q4_K_M (~16 Go, GPU-only), Qwen 2.5 9B Q8_0 (~9.8 Go, GPU-only).

🚀 Déploiement automatisé

Le script setup.sh orchestre l'installation complète depuis un Ubuntu fraîchement installé. Chaque étape est idempotente — relancer le script ne casse rien.

1
Clone & système
Clone le dépôt GitHub, met à jour Ubuntu, installe Docker et ROCm pour le GPU AMD.
2
Pare-feu UFW
Configure UFW : ports 22, 80, 443, 51820. Seul le trafic nécessaire est autorisé.
3
Secrets & mots de passe
Génère le .env avec tous les secrets aléatoires (openssl rand) et hash le mot de passe Authelia en bcrypt via le conteneur officiel.
4
Ollama bare metal
Installe Ollama, configure le service systemd (OLLAMA_HOST=0.0.0.0) et vérifie la détection automatique du GPU ROCm.
5
Stack Docker Compose
Lance les 7 services. Caddy émet automatiquement les certificats Let's Encrypt pour tous les sous-domaines de alba-arietis.com.
bash 
# Déploiement complet depuis zéro
git clone https://github.com/HugoLeonardi/llm-server.git
cd llm-server
bash setup.sh
